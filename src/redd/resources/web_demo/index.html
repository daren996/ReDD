Operator Guide

            
ReDD Operators

            

              ReDD turns document collections into queryable relational evidence through a small
              set of composable operators: dataset loading, schema generation, schema refinement,
              tuple extraction, filtering, and runtime optimization.
            

          


          

            

              Input
              
Dataset Loader

              
Reads registered datasets from the manifest contract and exposes documents, schema, queries, and ground truth.

            

            

              Schema
              
Preprocessing

              
Builds a query-independent schema view over the corpus and prepares reusable schema artifacts.

            

            

              Schema
              
Schema Refinement

              
Specializes the global schema for query-specific tables and attributes before extraction.

            

            

              Extraction
              
Data Extraction

              
Runs tuple extraction over selected datasets using generated or ground-truth schema sources.

            

            

              Optimization
              
Document Filtering

              
Prunes unlikely documents before expensive extraction while keeping the selected workflow intact.

            

            

              Optimization
              
Proxy Runtime

              
Uses lightweight proxy decisions from the active config to reduce oracle work where configured.

            

            

              Optimization
              
Adaptive Sampling

              
Selects informative documents for schema generation and refinement instead of processing uniformly.

            

            

              Optimization
              
Alpha Allocation

              
Coordinates risk and recall budget across filtering and extraction components.
